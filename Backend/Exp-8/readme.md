
## Learning Outcome
- I put together a Flask application with separate route files that handle both GET and POST, which really helped me understand how web frameworks and HTTP work.
- I set up a virtual environment and used a requirements.txt so anyone can recreate the project with the right dependencies.
- By breaking routes into their own modules, the code stayed clean and I got a feel for organizing a project like real developers do.
- I manually hit the APIs with curl/Postman to see how they behaved and learn how to verify responses.
- Writing the README reminded me how valuable clear docs are for anyone who needs to pick up the app later.
